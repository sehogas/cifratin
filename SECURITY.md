# Security Policy

## Supported Versions

Only the latest release of Cifratin is actively supported with security updates. We recommend always running the latest version available in the GitHub Releases.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of Cifratin seriously. If you find a security vulnerability, please do **NOT** open a public issue. Instead, follow these steps:

1. Send an email to the maintainer at **sehogas@gmail.com** with details of the vulnerability.
2. Include a description of the issue, step-by-step instructions to reproduce it, and any proof-of-concept (PoC) code if available.
3. We will acknowledge receipt of your report within 48 hours and work with you to resolve the issue promptly.

Please allow us reasonable time to investigate and patch the issue before making it public.
